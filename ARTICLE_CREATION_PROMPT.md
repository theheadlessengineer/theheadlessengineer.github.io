# Article Creation Prompt - Headless Engineer Blog

## Requirements
- **Word Count**: 2,000-5,000 words
- **Format**: Markdown with components
- **Audience**: Technical professionals, developers, architects
- **Tone**: Professional, data-driven

## Guardrails
**Prohibited**: Vendor promotion, outdated info (>12 months), unsubstantiated claims, copied content, speculation as fact
**Required**: 5+ credible citations, functional code, current best practices, source attribution

## Structure Template
```markdown
# [Title with Primary Keyword]

## Overview
- Problem/solution summary
- Key benefits with metrics

## What is [Core Concept]
<TechnicalTerm term="concept">[Definition]</TechnicalTerm>

## Implementation
### Step 1: [Action]
- Instructions + code examples
### Step 2: [Next Action]  
- Details + validation

## Performance & Benefits
- Quantified improvements with sources
- Business impact metrics

## Best Practices
- Standards + security + optimization

## Common Challenges
- Issues + solutions + troubleshooting

## FAQ
[5-8 questions with comprehensive answers]

## References
[Numbered source list]
```

## Citation Sources
Official docs, peer-reviewed papers, industry reports (Gartner/Forrester), authoritative tech blogs (Google/AWS), OSS repos, benchmarking studies

## Technical Standards
- Primary keyword in title/first paragraph
- Secondary keywords in H2/H3
- 3-5 internal links
- All code tested and functional
- TechnicalTerm components for key concepts
- Performance claims need benchmarks

## Validation Checklist
- [ ] 2,000-5,000 words
- [ ] 5+ credible citations  
- [ ] TechnicalTerm components used
- [ ] Functional code examples
- [ ] FAQ addresses search queries
- [ ] No prohibited content
